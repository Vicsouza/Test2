Testando as configurações do Git

Passos root-commit:

git init -> Para transoformar a pasta em um repositorio git
git status -> Para mostrar o status d repositorio
git add ("nome" ou "." -> para pegar todos os arquivos do repositorio) -> adiciona um arquivo para ser monitorado pelo git
git commit -m (-m serve para inserir uma mensagem relacionada a commit, ou seja, um "resumo" do que foi adicionado/alterado) -> Serve para realizar as alterações feitas no arquivo
Obs.: como é o primeiro commit (root-commit) os passos são esses;

Passos referente a alterações no arquivo

git add (nome) -> deve ser adicionado novamente o arquivo
git commit -m -> para realizar as alterações

O passo a passo desse arquivo aqui:

test -> git init test -> git status test -> git add index.html test -> git commit -m "adicionando os passos iniciais do arquivo de teste" test -> git status test -> git add index.html test -> git status test -> git commit -m "adicionado os passos restantes e o hitorico desse arquivo"

Testando as configurações do Git

Passos root-commit:

git init -> Para transoformar a pasta em um repositorio git
git status -> Para mostrar o status d repositorio
git add ("nome" ou "." -> para pegar todos os arquivos do repositorio) -> adiciona um arquivo para ser monitorado pelo git
git commit -m (-m serve para inserir uma mensagem relacionada a commit, ou seja, um "resumo" do que foi adicionado/alterado) -> Serve para realizar as alterações feitas no arquivo
Obs.: como é o primeiro commit (root-commit) os passos são esses;

Passos referente a alterações no arquivo

git add (nome) -> deve ser adicionado novamente o arquivo
git commit -m -> para realizar as alterações

Passos para criar e utilizar um servidor no git

mkdir serverTest -> criar uma pasta de servidor
git init --bare -> para indicar que esse é apenas um repositorio que contem as alterações dos arquivos e não os arquivos fisicos
git remote -> para mostrar todos os repositorios remotos relacionado repositorio
git remote add local "caminho do servidor"-> para adicionar um novo local remoto no repositorio master
git clone "endereço do servidor" -> serve para coletar os dados do servidor

Dados extras

git log -> para visualizar o historico de commit
.gitignore -> uma pasta que tem como finalidade informar quais arquivos devem ser ignorados pelo monitoramento do git
gitHub -> No canto superior direito no botão de "+", selecione "new repository"
git remote add "origin" "https://github.com/Vicsouza/Test.git" -> Função para adicionar o repositorio no git, as informações entre aspas podem ser alteradas
git push -u origin master -> para mandar os arquivos("-u" informar que esse seria o arquivo master, sendo necessario apenas o comando git push apartir dai)

Branches

git branch -> mostra as branches abertas do repositorio
git branch "nome" -> criar uma nova branch
git checkout "nome" -> mover para uma branch especifica
git checkout -b "nome" -> criar e mover para uma nova branch
git merge "nome" -> juntar a branch informada na que foi solicitado (ex: se estiver na master, e solicitar a branch "nome", a mesma se junta a master)
git rebase "nome" -> atualiza a branch em que está com as informações da branch informada

ctrl z

git checkout -- "nome" -> retorna as alteraçoes feitas antes de adiciona-las
git reset HEAD "nome" -> para voltar as alterações depois de ter executado o comando "git add"
git revert "hash" -> para reverter uma ação ja commitada
